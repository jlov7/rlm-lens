# RLM-Lens v0.1.0 — Public Demo Release

Release date: February 24, 2026  
Tag: `v0.1.0`

## Summary
v0.1.0 is the first public demo-ready release of RLM-Lens: a local-first analysis cockpit for evidence-backed answers, recursive trace inspection, and deterministic verification workflows.

## Highlights
- Guided onboarding with starter corpus packs for fast first-run experience.
- Evidence-backed answers with file and line-range citations.
- Interactive trace visualization (graph + timeline + node details).
- Operations deck: compare, watch, policy scan, and eval workflows.
- Multi-provider runtime controls and diagnostics:
  - OpenAI, Anthropic, Gemini, xAI, OpenRouter, Together, Groq, Fireworks.
- Hosted BYOK option:
  - Session key sent per run via request header.
  - Key is not persisted by application state/database.
- Deterministic visual verification across Chromium/Firefox/WebKit.

## Security and trust boundary notes
- Hosted BYOK keys are not persisted by RLM-Lens.
- Hosted backend/edge infrastructure still terminates requests and can observe headers in transit.
- For strongest trust guarantees, self-host the backend and use local environment variables for provider keys.

## Known limitations
- Provider execution compatibility depends on installed `rlm` backend support; unsupported combinations degrade to explicit fallback warnings.
- Token/cost accounting remains heuristic in fallback mode.
- Provider-native streaming token/cost telemetry is not yet complete.
